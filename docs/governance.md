## Защиты от injection
Основная часть работы агента будет без пользовательского ввода. Следовательно, риск промпт-инъекций сведен к нулю. 

Для выполнения "чувствительных" операций (постановка продукта в стоп, вызов сотрудников) будет необходимо подтверждение пользователя

## Логгирование
На этапе разработки MVP логирование работы системы будет выполняться при помощи библиотеки logger. Логироваться будет работа агента за каждый период его работы.

Данные не содержат такой персональных данных клиента, такие как платежные данные или ФИО. 

Информация о сотрудниках - будет использоваться только информация о графике. В LLM будет подаваться только количество сотрудников и график их работы, а отправка персональных данных в чат пользователя будет вне LLM.
Если возникнет необходимость использовать персональных данных сотрудника в контексте модели, данные будут маскироваться:
 - Иванов Иван -> Сотрудник 1

Так как анализ будет производиться раз в N минут, то логировать будем так же раз в N минут.

Пример логирования (еще будет дорабатываться):
```
09:00:00 Start Analysis id: 1
09:00:00 {cookingTime: 00:10:44, deliveryTime: 00:32:06, heatShelf: 00:03:21, employeeCount: 6, courierCount: 3 orderInWork: 2} # средние показатели за период N
09:00:01 размышлений модели
09:00:10 результат анализа
09:00:10 сообщение пользователю
```
