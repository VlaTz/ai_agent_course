## Защиты от injection
Основная часть работы агента будет без пользовательского ввода. Следовательно, риск промпт-инъекций сведен к нулю. 

Для выполнения "чувствительных" операций (постановка продукта в стоп, вызов сотрудников) будет необходимо подтверждение пользователя

## Riks register

| Риск                       | Описание                        | Последствия                                                                  | Вероятность | Защита                                          |
|:---------------------------|---------------------------------|------------------------------------------------------------------------------| ------------|-------------------------------------------------|
| Спам                       | Агент шлет в чат очевидные вещи | Пользователь не реагирует, и может пропустить действительно важные сообщения | Высокая| Предоставление контекста для анализа            |
| Галлюцинации               | Агент выдумал проблему          | "Шокирование" пользователя ложными данными                                   | Высокая| Разбиение задачи анализа на подзадачи           |
| Долгая обработка данных    | Агент долго обрабатывает данные | Момент реакции на проблему может быть упущен                                 | Средняя| Оптимизация частоты сбора и анализа показателей |

    
## Логгирование
На этапе разработки MVP логирование работы системы будет выполняться при помощи библиотеки logger. Логироваться будет работа агента за каждый период его работы
